## **POSIX — Portable Operating System Interface**

**POSIX** is a **standard** that defines how operating systems should expose certain APIs (Application Programming Interfaces) so that programs can run on multiple UNIX-like systems without major changes.

* Think of it as **a “contract”**: if an OS is POSIX-compliant, your program can rely on certain functions, constants, and behaviors.
* It mainly applies to **UNIX/Linux systems**, but macOS and even Windows (via some compatibility layers) support parts of POSIX.

---

### **Key Things POSIX Defines**

1. **Threads**

   * Functions like `pthread_create`, `pthread_join`, `pthread_mutex_lock`, etc.
   * Lets programs do **concurrent programming** (multi-threading).

2. **Process Management**

   * `fork()`, `exec()`, `getpid()`, `wait()`, etc.
   * Work with **child processes**, system calls, and process IDs.

3. **File I/O**

   * `open()`, `read()`, `write()`, `close()`, `lseek()`
   * Lower-level file operations than standard C `fopen/fread/fwrite`.

4. **Signals**

   * Handling interrupts and events from the OS, e.g., `SIGINT`, `SIGTERM`.
   * Functions: `signal()`, `kill()`, `sigaction()`.

5. **Synchronization**

   * Mutexes, condition variables, semaphores.
   * For safe concurrent programming.

6. **Other Utilities**

   * Timers (`clock_gettime()`), sleeping (`nanosleep()`), memory mapping (`mmap()`), etc.

---

### **Why It Matters**

* **Portability:** Write code on Linux, and it mostly works on BSD, macOS, or Solaris.
* **Consistency:** Programs behave the same way on different POSIX-compliant systems.
* **Power:** Access low-level OS features that standard C library (`stdio.h`) doesn’t cover.

---

### **Example: POSIX Thread**

```c
#include <pthread.h>
#include <stdio.h>

void* hello(void* arg) {
    printf("Hello from a thread!\n");
    return NULL;
}

int main() {
    pthread_t t;
    pthread_create(&t, NULL, hello, NULL);  // POSIX thread creation
    pthread_join(t, NULL);                  // wait for thread
    return 0;
}
```

* `pthread_create` and `pthread_join` are **POSIX functions**, not standard C.

---

In short: **POSIX is a standard that gives C/C++ programmers access to system-level features in a portable way**, especially for **processes, threads, and synchronization**.
